# Minecraft AFK Bot — DonutSMP

بوت ماينكرافت AFK لسيرفر DonutSMP. يبقى واقف في مكانه بدون حركة، ويعيد الاتصال تلقائياً لو انطرد. يشتغل 24/7 على استضافة مجانية.

---

## المميزات

- يتصل بـ `play.donutsmp.net` بمصادقة مايكروسوفت
- يقف في مكانه بدون أي حركة (physics معطل) — DonutSMP ما يكتشفه
- يعيد الاتصال تلقائياً لو انطرد أو انفصل
- لوحة تحكم ويب تشمل: حالة البوت، كود المصادقة، إرسال أوامر، سجل الأوامر
- يحفظ توكن مايكروسوفت — ما يحتاج تسجل دخول مرتين
- يشتغل على Railway مجاناً

---

## التشغيل محلياً

### 1. ثبّت المكتبات

```bash
npm install
```

### 2. اضبط البيئة

انسخ `.env.example` إلى `.env` وغيّر الإيميل:

```
MC_USERNAME=your_email@example.com
PORT=3000
```

### 3. شغّل البوت

```bash
npm start
```

### 4. اربط حساب مايكروسوفت

أول مرة يشتغل، البوت يعطيك كود:

1. افتح `http://localhost:3000`
2. شوف الكود (مثلاً `ABCD1234`)
3. افتح [microsoft.com/link](https://www.microsoft.com/link)
4. دخّل الكود وسجل دخول بإيميلك
5. البوت يدخل تلقائياً ✅

التوكن ينحفظ في `.auth-cache/` — ما يحتاج كود مرة ثانية.

## الاستضافة على Railway (مجاني)

1. ارفع المشروع على GitHub (private أو public)
2. سجل في [railway.app](https://railway.app) بـ GitHub
3. **New Project** → **Deploy from GitHub** → اختر المشروع
4. اذهب لـ **Variables** وأضف:
   - `MC_USERNAME` = إيميلك
5. Railway يعين `PORT` تلقائياً
6. اضغط **Deploy**
7. شوف **Deploy Logs** — أول مرة يظهر كود مصادقة
8. افتح **microsoft.com/link** ودخّله
9. بعد كذا البوت يشتغل 24/7 بدون تدخل

## شرح الكود بالتفصيل

### DNS Fix (سطر 9)
```js
dns.setServers(['8.8.8.8', '8.8.4.4', '1.1.1.1']);
```
يفرض Node.js يستخدم Google/Cloudflare DNS بدل DNS الافتراضي. بعض شبكات الـ DNS الافتراضية ما تحل أسماء مايكروسوفت وسيرفرات ماينكرافت.

### MC_USERNAME Check (سطر 17-21)
لو المتغير ما موجود، البوت ما يكسر العملية فوراً (عشان Railway ما يعيد تشغيله في لوب). ينتظر 30 ثانية ثم يخرج — Railway يعيد تشغيله ويعطيه فرصة ثانية.

### Chunk Size Filter (سطر 27-30)
مكتبة protodef تطبع رسائل "Chunk size is X but Y was read" في console.log. هذا عيب معروف في minecraft-protocol مع إصدار 1.21.11. البايتات الزيادة تُتجاهل بأمان، بس الرسائل تزحم الـ logs. هذا الفلتر يمنعها.

### Auth Cache (سطر 33-34)
مجلد `.auth-cache/` يحفظ توكن مايكروسوفت. لما البوت يسجل دخول أول مرة، prismarine-auth يحفظ التوكن في ملف JSON. المرة الجاية البوت يستخدم التوكن المحفوظ بدون ما يطلب كود جديد.

### Bot State (سطر 37-46)
- `bot` — كائن البوت من mineflayer
- `isReconnecting` — يمنع إعادة الاتصال المتكررة
- `botEnabled` — المستخدم يقدر يوقف البوت من الـ Dashboard
- `authCode` / `authUrl` — كود المصادقة لما الجلسة تنتهي
- `cmdLog` — سجل الأوامر والردود (آخر 200 رسالة)

### scheduleReconnect (سطر 49-60)
يؤخر إعادة الاتصال بعدد ثواني محدد. يمنع تكرار المحاولة لو `isReconnecting = true`. الأسباب المختلفة لها تأخيرات مختلفة:
- طرد عادي: 12 ثانية
- "already online": 30 ثانية
- "max account": 60 ثانية

### createBot (سطر 63-197)
ينشئ البوت بالإعدادات:
- `version: false` — يكتشف إصدار السيرفر تلقائياً
- `auth: 'microsoft'` — مصادقة مايكروسوفت
- `checkTimeoutInterval: 60000` — يفحص الاتصال كل 60 ثانية (ما يفصل بسرعة)
- `physicsEnabled: false` — يعطل الفيزياء من البداية — البوت ما يتحرك أبداً

#### onMsaCode (سطر 89-96)
يُستدعى فقط لما التوكن المحفوظ منتهي أو ما موجود. يعطي كود يُدخل في microsoft.com/link.

#### spawn event (سطر 108-116)
يعطل `bot.physicsEnabled` مرة ثانية كضمان إضافي. البوت يقف في مكانه — ما يرسل position packets. keep-alive packets تُرسل تلقائياً من minecraft-protocol فالسيرفر ما يطرده.

#### kicked event (سطر 120-137)
يستخرج نص السبب من JSON قد يكون معقد. يحفظه في `lastKickReason` عشان `end` event يستخدمه.

#### end event (سطر 139-157)
يُستدعى لما البوت ينفصل. يعيد الاتصال بتأخير حسب سبب الطرد.

#### error event (سطر 159-181)
يعالج الأخطاء:
- **Auth errors** (expired, 403): يعيد الاتصال
- **Network errors** (ETIMEDOUT, AggregateError, fetch failed): يعيد الاتصال بعد 15 ثانية
- **أخطاء ثانية**: يعرض الرسالة فقط

#### chat/message events (سطر 183-197)
يسجل كل رسائل الشات والسيرفر في `cmdLog` (يظهر في الـ Dashboard) وفي console.

### Unhandled Rejection Guard (سطر 200-210)
prismarine-auth أحياناً يرمي rejected promises بدون catch. هذا يمنع العملية من الانهيار.

### Web Dashboard (سطر 233-371)
صفحة HTML واحدة تعرض:
- **حالة البوت** (أونلاين/أوفلاين/يحتاج مصادقة)
- **كود المصادقة** لو الجلسة منتهية
- **حقل إرسال أوامر** — تكتب أي أمر وتضغط Send
- **سجل الأوامر** — ▸ بنفسجي = أمر أرسلته، ◂ رمادي = رد السيرفر
- **أزرار Stop/Start**

السجل يتحدث كل 3 ثواني بدون إعادة تحميل الصفحة (polling `/log`).

## العيوب والمشاكل المعروفة

1. **التوكن ينتهي** — بعد فترة طويلة (أسابيع)، توكن مايكروسوفت ينتهي والبوت يطلب كود جديد. ما في طريقة لتجديد التوكن تلقائياً بدون تدخل المستخدم.

2. **الـ Dashboard ما محمي** — أي أحد يعرف الرابط يقدر يوقف البوت ويرسل أوامر. لو حابب تحميه، أضف basic auth middleware.

3. **سجل الأوامل في الذاكرة** — `cmdLog` ينحفظ في RAM. لو البوت يعاد تشغيله، السجل يروح. لو تبي سجل دائم، اكتبه في ملف.

4. **لا دعم لعدة حسابات** — البوت يدعم حساب واحد فقط. لو تبي أكثر، لازم تشغل نسخ متعددة.

5. **physicsEnabled: false** — البوت ما يقدر يمشي أو يتفاعل مع العالم (يفتح صناديق، يضرب mobs، إلخ). هذا مقصود عشان DonutSMP ما يطرده.

6. **مشاكل DNS محلياً** — بعض الشبكات (خصوصاً في بعض الدول) ما تحل أسماء مايكروسوفت. على الاستضافة ما يكون فيه هذه المشكلة.

7. **لا تشفير بين البوت والسيرفر** — ماينكرافت بروتوكول ما يستخدم TLS. هذا عادي في ماينكرافت بس لازم تعرفه.

8. **Render/Koyeb الخطة المجانية** — Render ينام بعد 15 دقيقة بدون طلبات. Koyeb محدود بـ 512MB RAM. Railway أعطي $5 مجاني/شهر.
